# 振り返り — Reading & Task Board（10日間カリキュラム）

## わかったこと（5つ）

1. **フロントと API は役割分担が大事**  
   Next.js が画面、FastAPI がデータ処理、MySQL が保存。`fetch` でつなぐと、画面を変えても API をそのまま使える。

2. **GitHub は「履歴」だけでなく「開発の流れ」**  
   ブランチ → PR → マージ → Issue を close することで、一人でもチーム開発と同じ作法を体験できた。

3. **環境変数で設定を分離できる**  
   `.env` / `.env.local` にパスワードや API URL を置くと、コードを公開しても秘密情報を守れる。

4. **React の hooks でロジックを整理できる**  
   `useTasks` や `useBooks` に API 呼び出しをまとめると、画面コンポーネントが読みやすくなった。

5. **Cursor は「説明しながら作る」学習に向いている**  
   エラー文を貼ったり、カリキュラムを渡したりすると、次に何をすればよいか迷いにくい。

## まだ難しいこと（3つ）

1. **非同期処理の流れ**  
   `async/await`、`useEffect`、ローディング状態の切り替えがいつ起きるか、まだ頭の中で追いきれないことがある。

2. **SQL と DB 設計**  
   テーブル設計、外部キー、JOIN、インデックスなど、規模が大きくなると判断が難しそう。

3. **TypeScript の型**  
   API のレスポンス型や `unknown` の扱いなど、エラーメッセージを読んで自分で直すのに時間がかかる。

## 次に学びたいこと

- **認証** … ログインして自分のデータだけ見えるようにする（JWT やセッション）
- **テスト** … フロントのコンポーネントテスト、API の pytest
- **Docker** … `docker compose` で MySQL + Backend + Frontend を一発起動
- **デプロイ** … Vercel + クラウド DB など、ローカル以外で動かす方法
